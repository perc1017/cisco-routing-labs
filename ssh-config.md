# Настройка SSH на Cisco Router

## Что такое SSH

SSH (Secure Shell) — это протокол удаленного управления сетевыми устройствами.

В отличие от Telnet, SSH шифрует весь передаваемый трафик, включая:

* логины пользователей;
* пароли;
* выполняемые команды;
* вывод команд.

Поэтому в современных сетях рекомендуется использовать SSH вместо Telnet.

---

# Требования для работы SSH

Перед настройкой SSH на роутере должны быть выполнены следующие условия:

✅ Настроено имя устройства (`hostname`)

✅ Назначен IP-адрес хотя бы одному интерфейсу

✅ Настроен пароль режима Privileged EXEC (`enable secret`)

Если IP-адрес не настроен, подключиться к устройству удаленно будет невозможно.

---

# Проверка IP-адреса

Перед началом настройки убедимся, что интерфейс имеет IP-адрес.

```bash
R1# show ip interface brief
```

Пример вывода:

```text
Interface              IP-Address      OK? Method Status  Protocol
GigabitEthernet0/0     192.168.1.1     YES manual up      up
```

Статус `up / up` означает, что интерфейс включен как физически, так и на канальном уровне.

---

# Настройка доменного имени

Для генерации криптографических ключей Cisco IOS требует наличие доменного имени.

Устанавливаем его командой:

```bash
R1(config)# ip domain-name lab.local
```

Где:

* `lab.local` — произвольное доменное имя;
* можно использовать любое значение.

Проверить настройку можно командой:

```bash
R1# show running-config | include domain
```

---

# Создание локального пользователя

SSH использует локальную базу пользователей или внешний сервер аутентификации.

Создадим локальную учетную запись администратора:

```bash
R1(config)# username admin privilege 15 secret Cisco12345
```

### Описание параметров

| Параметр     | Назначение                           |
| ------------ | ------------------------------------ |
| admin        | Имя пользователя                     |
| privilege 15 | Максимальный уровень привилегий      |
| secret       | Хранение пароля в зашифрованном виде |
| Cisco12345   | Пароль пользователя                  |

---

# Генерация RSA-ключей

SSH использует RSA-ключи для шифрования соединения.

Создаем ключ длиной 2048 бит:

```bash
R1(config)# crypto key generate rsa modulus 2048
```

Пример вывода:

```text
% Generating 2048 bit RSA keys...
[OK]
```

После успешной генерации ключей SSH сможет использовать защищенное соединение.

---

# Включение SSH Version 2

После создания ключей рекомендуется использовать вторую версию SSH.

```bash
R1(config)# ip ssh version 2
```

Проверить можно командой:

```bash
R1# show ip ssh
```

Пример:

```text
SSH Enabled - version 2.0
```

---

# Настройка VTY-линий для SSH

Переходим к настройке виртуальных линий:

```bash
R1(config)# line vty 0 4
```

Указываем использование локальной базы пользователей:

```bash
R1(config-line)# login local
```

Разрешаем только SSH-подключения:

```bash
R1(config-line)# transport input ssh
```

Выходим из режима настройки:

```bash
R1(config-line)# exit
```

---

# Что делают эти команды

### login local

Заставляет устройство использовать локальную базу пользователей для аутентификации.

Например:

```bash
username admin privilege 15 secret Cisco12345
```

При подключении потребуется ввести именно эти учетные данные.

---

### transport input ssh

Разрешает подключения только через SSH.

После настройки:

✅ SSH работает

❌ Telnet заблокирован

Это значительно повышает безопасность устройства.

---

# Проверка настроек SSH

Проверяем состояние SSH:

```bash
R1# show ip ssh
```

Пример:

```text
SSH Enabled - version 2.0
Authentication timeout: 120 secs
Authentication retries: 3
```

---

# Проверка RSA-ключей

Убедимся, что ключи успешно созданы:

```bash
R1# show crypto key mypubkey rsa
```

Если ключ отображается в выводе команды, генерация выполнена успешно.

---

# Подключение через SSH

После завершения настройки можно подключиться к устройству удаленно.

### Linux

```bash
ssh admin@192.168.1.1
```

### macOS

```bash
ssh admin@192.168.1.1
```

### Windows PowerShell

```powershell
ssh admin@192.168.1.1
```

После подключения устройство запросит пароль:

```text
Password:
```

После успешной аутентификации откроется консоль Cisco IOS.

---

# Проверка активных SSH-сессий

Для просмотра активных SSH-подключений используйте:

```bash
R1# show ssh
```

Пример:

```text
Connection Version Mode Encryption Hmac State Username
0          2.0     IN   aes128     sha1 Session Started admin
```

---

# Полный пример настройки SSH

```bash
enable
configure terminal

hostname R1

ip domain-name lab.local

username admin privilege 15 secret Cisco12345

crypto key generate rsa modulus 2048

ip ssh version 2

line vty 0 4
 login local
 transport input ssh
 exit

end

copy running-config startup-config
```

---

# Проверка готовности устройства

После настройки рекомендуется выполнить следующие команды:

```bash
show ip interface brief
show ip ssh
show user
show ssh
```

Они позволят убедиться, что:

* интерфейсы работают корректно;
* SSH включен;
* пользовательские сессии отображаются;
* SSH-подключения принимаются устройством.

---

# Сохранение конфигурации

После завершения настройки сохраните конфигурацию:

```bash
R1# copy running-config startup-config
```

или из режима конфигурации:

```bash
R1(config)# do copy running-config startup-config
```
