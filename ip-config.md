# Настройка IP-адресов, масок подсети и шлюзов на Cisco Router

## Что такое IP-адрес, маска и шлюз

Перед настройкой интерфейсов необходимо понимать назначение основных сетевых параметров.

### IP-адрес

IP-адрес — это уникальный адрес устройства в сети.

Пример:

```text
192.168.1.1
```

---

### Маска подсети

Маска определяет, какая часть IP-адреса относится к сети, а какая к узлам.

Пример:

```text
255.255.255.0
```

Маска `255.255.255.0` соответствует префиксу:

```text
/24
```

---

### Шлюз по умолчанию (Default Gateway)

Шлюз используется для передачи трафика в другие сети.

Пример:

```text
IP-адрес ПК:     192.168.1.10
Маска:           255.255.255.0
Шлюз:            192.168.1.1
```

В данном случае адрес `192.168.1.1` принадлежит интерфейсу роутера, через который компьютер получает доступ к другим сетям.

---

# Просмотр интерфейсов роутера

Перед настройкой полезно посмотреть список интерфейсов.

Выполните команду:

```bash
R1# show ip interface brief
```

Пример вывода:

```text
Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0     unassigned      YES unset administratively down  down
GigabitEthernet0/1     unassigned      YES unset administratively down  down
```

Здесь видно, что интерфейсы пока не имеют IP-адресов и находятся в отключенном состоянии.

---

# Назначение IP-адреса интерфейсу

Перейдем в режим настройки интерфейса.

Пример настройки интерфейса GigabitEthernet0/0:

```bash
R1(config)# interface gigabitethernet 0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
```

### Описание команд

| Команда                                | Назначение                           |
| -------------------------------------- | ------------------------------------ |
| `interface gigabitethernet 0/0`        | Переход к настройке интерфейса       |
| `ip address 192.168.1.1 255.255.255.0` | Назначение IP-адреса и маски подсети |
| `no shutdown`                          | Включение интерфейса                 |

---

# Проверка настроек

После настройки можно убедиться, что интерфейс работает.

```bash
R1# show ip interface brief
```

Пример:

```text
Interface              IP-Address      OK? Method Status  Protocol
GigabitEthernet0/0     192.168.1.1     YES manual up      up
```

Статус `up / up` означает, что интерфейс включен как физически, так и на канальном уровне. Если отображается `administratively down`, интерфейс отключен командой `shutdown`.

---

# Настройка нескольких интерфейсов

Если роутер соединяет несколько сетей, каждому интерфейсу назначается свой IP-адрес.

Пример:

```bash
R1(config)# interface gigabitethernet 0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown

R1(config-if)# exit

R1(config)# interface gigabitethernet 0/1
R1(config-if)# ip address 10.0.0.1 255.255.255.0
R1(config-if)# no shutdown
```

После настройки получаем:

| Интерфейс | IP-адрес       |
| --------- | -------------- |
| G0/0      | 192.168.1.1/24 |
| G0/1      | 10.0.0.1/24    |

Теперь роутер может выполнять маршрутизацию между двумя сетями.

---

# Настройка шлюза на компьютере

На конечных устройствах шлюзом указывается IP-адрес интерфейса роутера, к которому они подключены.

Пример:

```text
IP-адрес ПК:     192.168.1.10
Маска:           255.255.255.0
Шлюз:            192.168.1.1
```

Где:

* `192.168.1.10` — адрес компьютера
* `255.255.255.0` — маска подсети
* `192.168.1.1` — адрес роутера (шлюз по умолчанию)

При попытке отправить трафик в другую сеть компьютер будет направлять его на указанный шлюз.

---

# Настройка шлюза на коммутаторе

Если используется управляемый коммутатор уровня L2, ему можно назначить шлюз для удаленного управления.

```bash
Switch(config)# ip default-gateway 192.168.1.1
```

Коммутатор будет использовать указанный адрес для доступа к устройствам за пределами своей локальной сети.

---

# Проверка связности

После настройки рекомендуется проверить работу сети.

### Проверка интерфейсов

```bash
R1# show ip interface brief
```

### Проверка таблицы маршрутизации

```bash
R1# show ip route
```

### Проверка доступности узла

```bash
R1# ping 192.168.1.10
```

Если ответы получены успешно, базовая настройка выполнена корректно.

---

# Сохранение конфигурации

По умолчанию все изменения сохраняются только в оперативной памяти (**RAM**).

После перезагрузки устройства настройки будут потеряны.

Чтобы сохранить текущую конфигурацию в энергонезависимую память (**NVRAM**), выполните:

```bash
R1# copy running-config startup-config
```

или из режима конфигурации:

```bash
R1(config)# do copy running-config startup-config
```

### Что происходит

* `running-config` — текущая конфигурация в RAM.
* `startup-config` — конфигурация, которая загружается при включении устройства.
* Команда копирует текущие настройки в постоянную память.

---

# Итоговая последовательность команд

```bash
enable
configure terminal

interface gigabitethernet 0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown
 exit

interface gigabitethernet 0/1
 ip address 10.0.0.1 255.255.255.0
 no shutdown
 exit

do copy running-config startup-config
```

После выполнения данных действий роутер будет:

✅ Иметь настроенные IP-адреса на интерфейсах

✅ Работать в качестве шлюза для подключенных устройств

✅ Выполнять маршрутизацию между сетями

✅ Отвечать на запросы `ping`

✅ Сохранять настройки после перезагрузки
